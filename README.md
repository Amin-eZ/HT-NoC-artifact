# HT-NoC Artifact — Reproducibility Package

## Overview

This artifact provides a **cycle-accurate simulator** and scripts to reproduce results from Sections **6.2** and **6.3** of the paper.

It includes:

- A simulator modeling **HT-NoC** and **baseline NoC**
- Configuration files for all evaluated workloads
- Scripts to reproduce the results reported in the paper
- Support for custom experiments on FC, CONV, DW, PW, and FFN workloads

Although some results in the paper were originally obtained using **RTL simulation**, the provided simulator reproduces identical latency results based on the same analysis.

---

## Notes on Section 6.2 (RTL vs Simulator)

- The paper reports latency results obtained using **RTL simulation**
- The RTL implementation is **not publicly available**
- The provided simulator is derived from the same analysis

The artifact reproduces latency results.

The artifact does **not** include:
- RTL implementation
- synthesis results (area / timing)
- power or energy measurements

---

## Note on Section 6.3.2

- Section 6.3.2 includes only results generated by this artifact. Results corresponding to related work are not reproduced.

## Running Custom Experiments

The artifact supports custom configurations for:

- FC layers
- CONV layers
- DW layers
- PW layers
- FFN blocks

### General Usage

```bash
python3 src/experiments/<script>.py \
    --config <your_config.json> \
    --output-dir <output_directory>
```

### Examples

#### Custom FC

```bash
python3 src/experiments/run_section_6_2_2_fc.py \
    --config configs/templates/fc_template.json \
    --output-dir outputs/custom_fc
```

#### Custom CONV

```bash
python3 src/experiments/run_section_6_2_3_conv.py \
    --config configs/templates/conv_template.json \
    --output-dir outputs/custom_conv
```

#### Custom DW

```bash
python3 src/experiments/run_section_6_2_4_dw.py \
```

#### Custom PW

```bash
python3 src/experiments/run_section_6_2_5_pw.py \
```

## Important Implementation Details

### Padding

All convolution and depthwise layers use:

```text
padding = 1
```

Therefore:

```text
ifmap_side = ifmap_side_original + 2
```

Example:

- Original: `224 × 224`
- Simulated: `226 × 226`

This matches the values used in the paper.
